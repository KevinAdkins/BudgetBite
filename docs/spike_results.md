# Spike Results

1. **CNN performance metrics**
The computer vision model is not performing as we hoped and was very inaccurate, something we are fixing with our pivot.

2. **Identified limiations**
  We tried using a computer vision segmentation model, but it was very inaccurate and couldn't be trained as we wanted.

3. **Rationale for pivot**
  We moved over to an LLM for better accuracy and easier use.

4. **Updated architecture plan**
  Use an LLM model and a free database we can pull recipes, ingredients, and pictures from.

5. **Risk mitigation plan**
  The computer vision model was risky to use since we didn't have the data or the time to use it. <br>
  We mitigated to an LLM for vision to text since that was beneficial in our time constraints.
